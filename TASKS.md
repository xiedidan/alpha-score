## 📝 任务操作指南

### 1️⃣ 查看任务
```bash
# 查看全局任务列表
cat TASKS.md

# 查看详细任务描述
cat tasks/details/todo/001-init-backend.md
```

### 2️⃣ 认领任务
1. 在 TASKS.md 中将 `@无人认领` 改为 `@你的名字`
2. 更新顶部的"Agent工作分配"表格
3. 立即提交避免冲突：
```bash
git pull origin main
git add TASKS.md
git commit -m "task: 认领任务 #001"
git push origin main
```

### 3️⃣ 开始任务
1. 将任务从 TODO 移动到 DOING
2. 移动详情文件：`mv tasks/details/todo/001-*.md tasks/details/doing/`
3. 在任务中添加 `开始:2025-11-27 10:00`
4. 提交变更

### 4️⃣ 完成任务
1. 将 `[ ]` 改为 `[x]`
2. 移动任务到 DONE 区域
3. 移动详情文件：`mv tasks/details/doing/001-*.md tasks/details/done/`
4. 添加 `完成:日期` 和 `⏱️ 实际:工时`
5. 提交变更并推送

### 5️⃣ 遇到阻塞
1. 移动任务到 BLOCKED 区域
2. 添加阻塞原因
3. 及时通知团队（人类决策者）

## ⚠️ 重要规则（所有AI Agent必须遵守）

### 🚨 强制规则
1. **开始工作前必须先 `git pull`**
2. **认领任务立即提交，不要拖延**
3. **一次只能认领一个任务**（除非有明确许可）
4. **禁止修改他人已认领的任务**
5. **完成任务必须更新TASKS.md和详情文件**
6. **遇到问题立即标记为BLOCKED并说明原因**

### 📋 任务规范
- 任务编号从 #001 开始递增
- 任务标题简洁明了（不超过50字）
- 必须标注优先级（高🔴/中🟡/低🟢）
- 必须标注预计工时和依赖关系
- 复杂任务必须创建详情文件

### 🔄 同步规范
- 每次操作前：`git pull`
- 每次操作后：`git add TASKS.md && git commit -m "task: xxx" && git push`
- 提交信息格式：`task: <操作> #<任务编号> <简短描述>`
  - 示例：`task: 认领 #001 后端初始化`
  - 示例：`task: 完成 #001 后端初始化`
  - 示例：`task: 更新 #001 添加依赖安装步骤`

- 遇到冲突时人工解决，不要强制推送
